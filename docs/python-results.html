Reading notes

What the scorer counts

  • A axes (anti-fragility, lower better): A.1 substring asserts, A.2 private-symbol use, A.4 recomputed crypto, A.5 or-joined matches.
  • B.1 fixed-vector asserts (higher better) · C.1 real-mock LOC (lower).
  • D.1 LOC/test (lower = reuse) · D.2 parametrize ratio (higher).
  • Semantic axes (A.3/A.6/B.2/B.3/E.*) are not auto-counted — this tally is a floor, not the full verdict.

Why the deltas are trustworthy even if levels aren't

  • The auto-scorer is more generous than the coverage-driven hand audit (which judged ~1/9 truly better), so absolute tallies are soft.
  • But the same instrument scores all three arms, so the prompt-vs-model decomposition holds.
  • A.2 is the one axis the model moved more than the prompt (118 → 59); A.1/C.1/D.2 were solved by the prompt alone.
